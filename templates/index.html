How to correct the data on this dashboard

If an agreement is missing, an issue has been wrongly flagged, or any field is incorrect, you have three options:

  1. Ask Claude (easiest) — in Cowork or Claude Code, say e.g. "In the TUULIKARHU agreements database, mark the NDA with ACME Oy as reviewed" or "Add the missing service agreement filename.pdf with counterparty X, signed 2026-04-12". Claude will edit agreements_database.json on Google Drive and the dashboard will update on next page refresh.
  2. Dismiss a flag — click the dismiss button next to any flag in the "Needs attention" panel. The dismissal is saved to Google Drive.
  3. Edit the JSON directly — open agreements_database.json in Google Drive and edit the relevant entry. Refresh the page to see the change.

Adding new agreements: upload new PDFs to the relevant zip archive in Google Drive, then run python build_database.py locally to re-scan and refresh the database.

How this app works

The dashboard is hosted on Railway and accessible from any browser. Data lives in agreements_database.json on Google Drive — read on every page load, no rebuild needed. Access is protected by username and password.

Adding an agreement

Upload the signed PDF to the matching zip archive in Google Drive, then run python build_database.py locally to re-scan and push the updated database back to Google Drive. Refresh the dashboard to see the change.

Correcting metadata

Tell Claude: "In the TUULIKARHU database, change the expiry date of <agreement> to 2027-12-31". Logged in manual_corrections_log. Then refresh.

Access

Share the Railway URL with anyone who should have access. Credentials are managed via Railway environment variables (DASHBOARD_USERNAME / DASHBOARD_PASSWORD). The Google Drive JSON remains the single source of truth.

Attention rules

  • Payment due within 30 days
  • Term ending within 90 days
  • Renewal notice deadline within 60 days
  • Non-compete ending within 30 days
  • Loan maturity within 30 days
  • Status = "review" always flagged
  • Added in last 7 days shown on front page
  • Missing parties / signed_date / category flagged (Protocol category excluded)

API

All endpoints require Basic Auth (same credentials as the dashboard).
GET /api/data — full agreements_database.json
GET /api/file/<id> — serves the PDF for an agreement inline
POST /api/dismiss — body: {"agreement_id":"…","flag_type":"expired"}
POST /api/update — body: {"agreement_id":"…","field":"status","value":"terminated"}
GET /healthz — unauthenticated diagnostic endpoint (Drive connectivity check)
